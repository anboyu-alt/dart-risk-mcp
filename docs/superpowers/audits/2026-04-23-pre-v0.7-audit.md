## 3. 🔴 CRITICAL — 1건

### #1. `cb_extractor.py`가 HTML 파싱에 의존 — 구조화 엔드포인트로 교체 필요

**현재 구현:**
- `/api/document.xml` ZIP 다운로드 → HTML 텍스트 정규식 추출 → 인수인명 파싱
- 107줄 중 regex 6회, API 호출 1회 (document만)
- 취약점: 공시 포맷 변경·표 레이아웃 변화 시 추출 실패, 한글 인수인명에 특수문자 포함 시 오염

**권고:**
- `/api/cvbdIsDecsn.json` (전환사채 발행결정) 직접 호출
- `/api/bdwtIsDecsn.json` (신주인수권부사채 발행결정) 직접 호출
- `/api/exbdIsDecsn.json` (교환사채 발행결정) 직접 호출
- 각 엔드포인트의 **인수인 그룹 필드** 사용: `actsen`(인수인구분), `actnmn`(인수인명) — 이미 가이드에서 확인됨 (line 2431~2432, 2518~2519)

**영향 도구 (3개):**
- `find_actor_overlap` — 세력 추적 정확도 ↑ (v0.7.0 C안과 직결)
- `build_event_timeline` — CB 인수자 표시 신뢰성 ↑
- `analyze_company_risk` — CB 관련 복합 패턴 매칭 ↑

**마이그레이션 전략:**
- 기존 `extract_cb_investors(rcept_no)` 시그니처 유지
- 내부 구현만 구조화 엔드포인트 우선 시도 → 실패 시 HTML 파싱 폴백
- 테스트: 기존 유닛 + 구조화 응답 모킹 테스트 추가

## 4. 🟡 IMPORTANT — 5건

### #2. AUDIT 신호 강화 — 감사의견 구조화 엔드포인트

**현재:** `signals.py`가 "감사의견 비적정", "감사의견거절" 등 키워드 매칭
**권고 엔드포인트:**
- `/api/accnutAdtorNmNdAdtOpinion.json` (회계감사인의 명칭 및 감사의견)
- `/api/adtServcCnclsSttus.json` (감사용역체결현황)
- `/api/accnutAdtorNonAdtServcCnclsSttus.json` (비감사용역체결현황)

**효과:** 감사의견 종류(적정/한정/부적정/의견거절) 정확 분류, 감사인 교체 이력 정량화
**영향:** `check_disclosure_anomaly` 지표 ② "감사의견 이슈" 신뢰도 상승

### #3. CB_ROLLOVER/CB_REPAY 신호 강화 — 미상환 잔액 시계열

**현재:** 공시명 키워드 매칭만
**권고 엔드포인트 (5종):**
- `/api/entrprsBilScritsNrdmpBlce.json` (기업어음 미상환 잔액)
- `/api/srtpdPsndbtNrdmpBlce.json` (단기사채 미상환 잔액)
- `/api/cprndNrdmpBlce.json` (회사채 미상환 잔액)
- `/api/newCaplScritsNrdmpBlce.json` (신종자본증권 미상환 잔액)
- `/api/cndlCaplScritsNrdmpBlce.json` (조건부자본증권 미상환 잔액)

**효과:** 실제 잔액 추이로 롤오버 반복 정량 판정 가능
**영향:** `track_capital_structure` → CAPITAL_CHURN/CB_ROLLOVER 정확도 ↑

### #4. INSIDER 거래 강화 — 대량보유 변경 이벤트 엔드포인트

**현재:** 연도별 `hyslrSttus` 스냅샷 비교로 Δ 계산
**권고 엔드포인트:**
- `/api/hyslrChgSttus.json` (대량보유상황 변경 현황) — 가이드 line에서 확인
- `/api/tesstkAcqsDspsSttus.json` (자기주식 취득·처분 현황)

**효과:** 변경 시점 정확도 ↑ (연도 경계 누락 방지), 자사주 거래 동시 추적
**영향:** `track_insider_trading` 30일 클러스터 탐지 정확도 ↑

### #5. CAPITAL_CHURN 강화 — 증자/감자/자사주 결정 구조화 엔드포인트

**현재:** report_nm 키워드 매칭 (`REVERSE_SPLIT`, `TREASURY`, `3PCA` 등)
**권고 엔드포인트:**
- `/api/piicDecsn.json` (유상증자 결정) ← **v0.7.0 C안 이미 확정**
- `/api/fricDecsn.json` (무상증자 결정)
- `/api/pifricDecsn.json` (유무상증자 결정)
- `/api/crDecsn.json` (감자 결정)
- `/api/tsstkAqDecsn.json` (자기주식 취득 결정)
- `/api/tsstkDpDecsn.json` (자기주식 처분 결정)

**효과:** 수량·가액·목적 구조화 조회 → 증자/감자 대칭성 교차검증 → 희석률 계산 가능
**영향:** `track_capital_structure` / CAPITAL_CHURN 플래그 / `find_actor_overlap` 확장 (유상증자 인수자)

### #6. 재무이상 기준 보강 — 기업 지표 엔드포인트

**현재:** v0.6.1에서 `fnlttSinglAcntAll`로 전환, 절대값 YoY 비교만
**권고 엔드포인트:**
- `/api/fnlttSinglIndx.json` (단일 기업 재무지표)

**효과:** 업종 평균 대비 이상 탐지, 수익성·안정성 지표 직접 제공
**영향:** `scan_financial_anomaly` 임계값 재조정(A안) 시 업종 기준점 확보

## 8. 결론

v0.7.0은 "A + C + 🔴 #1" 3개를 묶어 진행.
🟡 2~6은 감사 결과로 확실히 인식되었으니 v0.8 스펙 작성 시 첫 후보로 올림.

> 이 감사는 공식 가이드(`opendart_api_guide.md`, 2749줄) 기준 82개 엔드포인트 전수 매칭 결과다. 구현 전 샘플 응답 검증이 추가로 필요하며, 실제 필드명·에러코드는 API 호출 테스트에서 확정한다.
