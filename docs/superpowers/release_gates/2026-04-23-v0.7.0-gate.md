**원인 분석:**
- `fetch_piic_decision(rcept_no, api_key)` 가 `piicDecsn.json`을 `rcept_no` 파라미터로 호출하나, DART API는 `corp_code + bgn_de + end_de` 조합을 요구함. API status 100 반환 → 빈 dict.
- `corp_code + 날짜범위`로 직접 조회 시 응답은 오나, `actnmn` 필드가 `null` — 주주배정·일반공모 유상증자는 특정 인수인명이 없음(공모 대상 불특정 다수).
- 소수 사모 배정 유상증자(`ssl_at="주주배정" 외 유형`)는 actnmn 필드가 채워질 수 있으나, 최근 90일 샘플 3사는 모두 해당 없음.

**교정 제안(v0.7.1):**
1. `fetch_piic_decision` / `fetch_pifric_decision` 의 파라미터를 `rcept_no` → `corp_code + bgn_de + end_de`로 수정.
2. `ssl_at`(배정방법) 필드로 사모 배정 여부 판별 후 `actnmn` 추출 조건부 시도.

### G4 CB 구조화 추출 검증

대상 공시: 아이티센피엔에스, rcept_no=`20260422000707`, 주요사항보고서(전환사채권발행결정)

**구조화 경로 확인 (monkey-patch):**

```python
# _legacy_html_extract 를 RuntimeError 발생 함수로 교체 후 호출
_cbmod._legacy_html_extract = lambda r, a: (_ for _ in ()).throw(RuntimeError("fallback invoked"))
investors = _cbmod.extract_cb_investors("20260422000707", api_key)
# → 정상 반환 (RuntimeError 미발생) = 구조화 경로 확정
```

폴백 미호출 확인 → 구조화 경로 동작 **확정**.
그러나 인수자 목록: **0건** → G4 FAIL (PASS 조건: 구조화 AND 인수자 비어 있지 않음)

**원인 분석:**
- `fetch_cb_issue_decision(rcept_no, api_key)` → `cvbdIsDecsn.json?rcept_no=…` 호출
- DART API는 `cvbdIsDecsn.json`에 `corp_code + bgn_de + end_de` 필요, `rcept_no`만으로는 status 100(필수값 누락) 반환 → 빈 dict → `_parse_structured({})` → 빈 리스트 → 폴백으로 진행하지 않음(structured가 아무것도 반환 안 해도 "성공적으로 빈 응답"으로 처리됨)
- HTML 경로: 최근 CB 공시 6건 모두 인수인이 "미정"(추후 결정) 또는 쿨옵션 계약 문맥에서만 언급 — 실제 인수인명 없음
- 결론: API 파라미터 버그(rcept_no → corp_code+date)로 인해 구조화 엔드포인트가 사실상 항상 빈 응답. 폴백도 최근 공시에서는 효과 없음.

**교정 제안(v0.7.1):**
1. `fetch_cb_issue_decision` / `fetch_bw_issue_decision` / `fetch_eb_issue_decision` 에 `corp_code + bgn_de + end_de` 파라미터 전달 방식으로 변경.
2. `cb_extractor.extract_cb_investors` 시그니처에 `corp_code` + 날짜 범위 인자 추가 또는 `resolve_corp` 결과를 서버에서 전달.
3. HTML 폴백: "미정" 외에 인수인 절 내 법인명 패턴(주식회사/㈜/펀드/조합/파트너스) 강화.
