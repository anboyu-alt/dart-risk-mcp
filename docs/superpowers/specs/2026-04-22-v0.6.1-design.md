## 1. 배경 및 문제 정의

### V.2 릴리스 게이트 결과 (2026-04-22)

| 기업 | CAPITAL_CHURN | 재무이상 | 결과 |
|------|--------------:|---------:|------|
| 5개 FSS 사례 (v0.2) | 3/5 탐지 | **0/5 탐지** | 게이트 통과(최소조건) but 재무이상 완전 미작동 |
| 삼성전자 | **탐지됨 (거짓양성)** | — | 대형주 자사주 매입만으로 플래그 |

### 핵심 문제

- **A. 거짓양성**: `CAPITAL_EVENT_KEYS`가 9개 신호를 동일 취급. 정상 기업의 자사주 매입·소각(`TREASURY`)만으로도 ≥3건 쉽게 충족.
- **B-1. 탐지 누락**: `AR_SURGE`/`INVENTORY_SURGE` `delta >= 50`, `CAPITAL_IMPAIRMENT` `ratio < 50` 등 임계값이 과도하게 보수적. V.2 5개 기업 모두 미탐지.
- **B-2. 패턴 미연동**: v0.6.0 신규 신호 5개가 기존 복합 패턴 8개에 반영되지 않아 composite 판정에서 누락.

## 2. 설계 결정

### A. CAPITAL_EVENT_KEYS 이원화 (A-1)

자본 이벤트를 **희석성(dilutive)** 과 **비희석성(non-dilutive)** 으로 분리:

```python
# 희석성 — 주주가치 훼손 우려
DILUTIVE_CAPITAL_EVENTS = frozenset({
    "CB_BW", "EB", "RCPS",
    "3PCA", "RIGHTS_UNDER",
    "CB_ROLLOVER", "GAMJA_MERGE",
})

# 비희석성 — 정상 기업도 반복 수행
NON_DILUTIVE_CAPITAL_EVENTS = frozenset({
    "TREASURY", "CB_BUYBACK", "TREASURY_EB",
})

# 하위 호환 (전체 집합)
CAPITAL_EVENT_KEYS = DILUTIVE_CAPITAL_EVENTS | NON_DILUTIVE_CAPITAL_EVENTS
```

**판정 규칙 변경** (`detect_capital_churn`):

| 조건 | 플래그 |
|------|-------:|
| 12개월 내 희석성 ≥ 3건 | **CAPITAL_CHURN** |
| 또는 희석성 ≥ 2건 AND 비희석성 ≥ 2건 | **CAPITAL_CHURN** |
| 비희석성만 ≥ 3건 | — (정상) |

**예상 효과**: 삼성전자 자사주 매입 단독 시나리오 → 거짓양성 제거. 기존 V.2 3/5 탐지 유지(희석성 중심 사례).

### B-1. 재무이상 임계값 경험적 재조정 (B-1b)

V.2 5개 기업의 실제 FS 데이터를 조회해 delta 분포를 분석하고 **2~3건 탐지 가능** 수준으로 임계값 재설정.

**절차** (구현 중 1회 실행):
1. V.2 5개 기업(code_list) × 당기/전기 FS 조회
2. 각 기업의 `매출채권Δ%`, `재고자산Δ%`, `영업현금흐름/순이익` 비율, `(자본-자본금)/자본금*100` 측정
3. 분포를 2nd 사분위 이상 포착하는 임계값 선택 (예상 범위):
   - `AR_SURGE`: `delta >= 50` → **`delta >= 30`**
   - `INVENTORY_SURGE`: `delta >= 50` → **`delta >= 30`**
   - `CASH_GAP`: (현행 유지 예상)
   - `CAPITAL_IMPAIRMENT`: `ratio < 50` → **`ratio < 80`**

**폴백**: V.2 데이터로 3/5 탐지가 수학적으로 불가능하면 사용자에게 알리고 중단.

### B-2. 복합 패턴 2개 확장 (B-2 Option 2)

```python
# taxonomy.py CROSS_SIGNAL_PATTERNS

"zombie_ma": {
    ...
    "signal_sequence": [..., "2.7"],  # CAPITAL_CHURN 추가
},

"delisting_evasion": {
    ...
    "signal_sequence": [..., "2.7", "8.2"],  # CAPITAL_CHURN, CAPITAL_IMPAIRMENT 추가
},
```

나머지 6개 패턴은 손대지 않음 (v0.6.0 신호와 의미적 관련성 낮음).
