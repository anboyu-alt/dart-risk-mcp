## 2. 설계 결정

### A. 재무이상 임계값 경험적 재조정

#### A-1. 샘플 수집 전략

v0.6.1의 5개 샘플 제약을 다음과 같이 확장한다.

1. **모수 확보**: `search_market_disclosures(preset='all_risk', days=365, max_results=200)` 호출로 최근 12개월 재무이상 의심 후보 탐색
2. **필터링**: 상장폐지 기업 제외 — `resolve_corp()` 성공하고 `fnlttSinglAcntAll(year=직전, report_type='annual', fs_div='CFS')`가 당기·전기 모두 유효 데이터를 돌려주는 기업만 선별
3. **샘플 크기**: 최종 **10~20개** 확보 목표. 미달 시 `search_market_disclosures` 조회 기간을 24개월로 확장
4. **추가 제외 기준**: 당기 또는 전기에 `매출액 == 0` / `자본금 == 0` / 자산총계 결측인 비정상 값 제거

#### A-2. 임계값 결정 방법

각 샘플 기업에 대해 4개 지표를 실측:

| 지표 | 계산식 | 현재 임계값 |
|------|--------|------------|
| AR_SURGE | `(당기 매출채권 - 전기 매출채권) / 전기 매출채권 × 100` | ≥ 50% |
| INVENTORY_SURGE | `(당기 재고자산 - 전기 재고자산) / 전기 재고자산 × 100` | ≥ 50% |
| CASH_GAP | `영업활동현금흐름 / 당기순이익` (순이익 ≠ 0) | < 50% |
| CAPITAL_IMPAIRMENT | `자본총계 / 자본금 × 100` | < 50% |

**임계값 결정 규칙**: 각 지표의 분포에서 **2사분위수(중앙값)** 을 후보로 두고, 해당 임계값으로 재실행 시 샘플 중 **30~50%** 가 플래그로 포착되도록 조정(과민·둔감 모두 배제).

**보조 교차검증**: `fnlttCmpnyIndx` (다중 기업 재무지표) 사용해 업종 평균 대비 이상도 참고. 단, v0.7.0에서는 `detect_financial_anomaly` 자체 로직에는 업종 보정을 넣지 않는다(YAGNI). 업종 기준은 **임계값 결정의 참고 자료**로만 활용.

#### A-3. 기록

선정 근거를 `tmp/thresholds_v0.7_decision.md` 에 저장:
- 수집된 N개 기업 목록
- 지표별 분포(최소/중앙값/최대)
- 채택 임계값과 이유
- 업종 평균 교차검증 결과

#### C-3. `find_actor_overlap` 내부 로직

`server.py`의 `find_actor_overlap` 공개 시그니처는 **그대로**.

내부 변경:
1. 기업별 최근 365일 공시 목록에서 CB/BW/EB 공시 → `extract_cb_investors` 호출 (기존)
2. **신규**: 기업별 유상증자·유무상증자 공시 → `extract_rights_offering_investors` 호출
3. 통합: 인수인별 `[(company, source, amount, rcept_no), ...]` 누적
4. 2개 이상 기업에 등장하는 인수인만 "공통 행위자"로 출력
5. 반환 텍스트에 `source` 태그 표시 (`[CB]` / `[유상증자]`)

공시 조회 한도: 기존 CB 최대 3건/기업 → **CB + 유상증자 최대 5건/기업**으로 확대.

### 🔴 #1. `cb_extractor.py` 구조화 전환

#### D-1. 엔드포인트 dispatch 로직

```python
def extract_cb_investors(rcept_no: str, api_key: str) -> list[dict]:
    # 1. 공시 제목·내용에서 종류 추정 불가하므로 3개 엔드포인트를 순차 시도
    for fetch_fn in [fetch_cb_issue_decision, fetch_bw_issue_decision, fetch_eb_issue_decision]:
        try:
            data = fetch_fn(rcept_no, api_key)
            investors = _parse_investors_from_structured(data)
            if investors:
                return investors
        except Exception:
            continue
    # 2. 모든 구조화 엔드포인트 실패 시 HTML 폴백
    return _legacy_html_extract(rcept_no, api_key)
```

- 3개 엔드포인트 중 먼저 non-empty 결과를 돌려주는 것 채택
- 모든 구조화 조회가 실패하면 기존 HTML 파싱(`_legacy_html_extract`) 실행 → **하위호환 보장**

#### D-2. 반환 형식

기존과 동일:
```python
[{"name": "OO투자조합", "type": "투자조합", "amount": "10000000000"}, ...]
```

`source` 키는 **추가하지 않음** (기존 호출부 호환). `extract_rights_offering_investors` 는 `source` 키 포함 — 차이는 의도적.
