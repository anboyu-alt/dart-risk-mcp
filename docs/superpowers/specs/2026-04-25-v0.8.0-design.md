# v0.8.0 설계 — 탐지 정확도 업그레이드

> **생성일:** 2026-04-25
> **버전:** 0.7.5 → 0.8.0 (MINOR)
> **테마:** 키워드 매칭 기반 일부 탐지를 DART OpenAPI 구조화 엔드포인트로 대체해 정확도 상승.

---

## 1. 목표(Goal)

v0.7.x 시리즈가 "쉬운 출력"(사용자에게 보이는 문구 품질)을 완성했다면, v0.8.0은 **"탐지 정확도"**로 테마를 이어간다. pre-v0.7 감사(`docs/superpowers/audits/2026-04-23-pre-v0.7-audit.md`)에서 🟡로 이월된 5개 항목 중 구현 난이도·사용자 체감 가치·기존 도구 시너지가 가장 큰 2개(#2 AUDIT, #3 CB_ROLLOVER)를 선행 반영한다.

동시에 v0.7.x 동안 쌓인 단일 출력 원칙(expert mode 분기 폐기)을 문서로 공식 박는다.

### 2.2 #2 AUDIT 감사의견 구조화

**신규 DART 엔드포인트 3개 통합:**
- `/api/accnutAdtorNmNdAdtOpinion.json` — 감사인 명칭·감사의견(연도별)
- `/api/adtServcCnclsSttus.json` — 감사용역 체결현황(보수·용역시간)
- `/api/accnutAdtorNonAdtServcCnclsSttus.json` — 비감사용역 체결현황

**신규 핵심 함수:**
```python
# dart_client.py
def fetch_audit_opinion_history(
    corp_code: str,
    api_key: str,
    lookback_years: int = 5,
) -> dict:
    """3개 엔드포인트 통합. 연도별 감사의견·감사인·보수 dict 반환.

    반환 예:
        {
            "opinions": [{"year": 2025, "opinion": "적정",
                          "auditor": "삼일회계법인", "tenure_years": 3,
                          "audit_fee_okwon": 820_000_000,
                          "non_audit_fee_okwon": 110_000_000}, ...],
            "auditor_changes": [{"from_year": 2023, "to_year": 2024,
                                 "from": "한영", "to": "삼일",
                                 "reason": "자율 교체"}],
            "independence_warnings": ["2022 비감사용역 비중 32%"],
        }
    """
```

**신규 MCP 도구 `get_audit_opinion_history(company_name, lookback_years=5)`:**
- 위 함수를 한글 서술로 렌더
- 감사의견 한정·부적정·의견거절 연도 ⚠ 표시
- 감사인 교체 2회 이상이면 경고
- 비감사용역 비중 > 30%이면 독립성 경고

**기존 `check_disclosure_anomaly` 지표 ② 보강:**

| 조건 | 기존 점수 | v0.8.0 점수 |
|---|---|---|
| 감사의견 한정 | +5 (키워드) | +5 (구조화) |
| 감사의견 부적정 | +10 (키워드) | +10 (구조화) |
| 감사의견 거절 | +15 (키워드) | +15 (구조화) |
| **감사인 교체 3년 내 2회↑** | (탐지 불가) | **+5 (신규)** |
| **비감사용역 비중 >30%** | (탐지 불가) | **+3 (신규)** |

**Fallback:** `fetch_audit_opinion_history` 실패 시 기존 키워드 매칭 유지 — 사용자는 기능 열화를 감지하지 못한다.

### 2.3 #3 CB_ROLLOVER 미상환 잔액 시계열

**신규 DART 엔드포인트 5개 통합:**
- `/api/entrprsBilScritsNrdmpBlce.json` — 기업어음
- `/api/srtpdPsndbtNrdmpBlce.json` — 단기사채
- `/api/cprndNrdmpBlce.json` — 회사채
- `/api/newCaplScritsNrdmpBlce.json` — 신종자본증권
- `/api/cndlCaplScritsNrdmpBlce.json` — 조건부자본증권

**신규 핵심 함수:**
```python
# dart_client.py
def fetch_debt_balance(
    corp_code: str,
    api_key: str,
    year: str = "",
) -> dict:
    """5개 엔드포인트 통합. 채무증권 종류별 미상환 잔액·만기 구간 dict.

    반환 예:
        {
            "year": 2025,
            "by_kind": {
                "corporate_bond": {"total": 56_700_000_000,
                                    "maturity_under_1y": 8_900_000_000,
                                    "maturity_1_3y": 47_800_000_000},
                "short_term_bond": {"total": 2_300_000_000, ...},
                ...
            },
            "total": 72_300_000_000,
            "equity_ratio": 0.603,  # 자기자본 대비, 재무제표 재사용
            "maturity_1y_share": 0.23,
        }


def detect_debt_rollover(
    balances: list[tuple[int, int]],  # [(year, total_balance), ...]
    events: list[dict],                # 자본 이벤트 (기존 format)
) -> str | None:
    """3년 연속 잔액 YoY 변동 <10% AND 같은 기간 CB/BW 신규 발행 ≥2건 →
    'CB_ROLLOVER' 플래그 반환. 조건 미충족 시 None."""
```

**신규 MCP 도구 `track_debt_balance(company_name, year="")`:**
- 5개 채무증권 종류별 잔액 + 만기 구간 표 한글 렌더
- `year=""` 이면 직전 사업연도
- 총 미상환/자기자본 비율 > 50% ⚠ 표시
- 1년 내 만기 비중 > 30% ⚠ 표시

**기존 `track_capital_structure` 보강:**
- 자본 이벤트 목록 뒤에 `━━ 현재 미상환 채무증권 ━━` 블록 3~4줄 추가
- `detect_debt_rollover` 조합 판정으로 `CB_ROLLOVER` 플래그 탐지 정확도 상승(기존은 공시명 키워드 매칭만)

## 5. 데이터 흐름

```
(신규) fetch_audit_opinion_history ──┬── get_audit_opinion_history (신규 도구)
                                      └── check_disclosure_anomaly 지표 ② (보강, fallback 유지)

(신규) fetch_debt_balance ───────────┬── track_debt_balance (신규 도구)
                                      └── track_capital_structure 블록
                                              └── detect_debt_rollover → CB_ROLLOVER 플래그 고도화
```

**에러 처리 원칙(전 구간 공통):**
- API 403/404 → 빈 리스트 반환, 도구 레벨 예외 전파 없음.
- 부분 실패(5개 중 2개 실패) → 가능한 부분만 표시, 사용자에게 "일부 데이터 누락" 주석.
- 신규 엔드포인트 전체 실패 → 기존 키워드 매칭으로 fallback(지표 ②).

## 8. 설계 원칙(v0.7 계승 + 확정)

1. **내부 코드는 출력 경계를 넘지 못한다.** (신규 엔드포인트 원본 코드명 노출 금지)
2. **모든 수치에는 의미를 동반한다.** (예: `723억`이 아니라 `723억 (자기자본 60%)`)
3. **각 도구 출력은 맨 위 3~4줄로 독립적으로 읽힌다.**
4. **단일 출력 — level/mode 파라미터 분기 없음.** (v0.8.0에서 공식 확정)
